Features

Front-End Changes

Login Page

  • Description: A page for the user to login to an already existing account. Also has links to create account and forget password pages
  • Status: Working

Sign Up Page

  • Description: The page for the user to create an account with an email, password, name, and security question answer
  • Status: Working

Change Password Page

  • Description: The page that the user can change the password to their account aslong as they get the security question right
  • Status: Working

Create Event Form

  • Description: The form that the user can create events with. Asked for info such as title, start time, end time, date, and description. Appears when the user clicks on a specific day on the calender
  • Status: Working

View Event Info

  • Description: This section displays all the information that the database has on the event. Appears when the user clicks on the event in the calender. This place also gives the user the option to delete an event
  • Status: Working

Calender

  • Description: Made it where the events that the user created display on the proper date in the calender. Also I made the calender go all the way to current dates.
  • Status: Working

Back-End Changes

Accounts Table Database

  • Description: Database that holds user information such as id, name, email, password, account creation time, last login time, number of logins, security question answer, and failed login attempts
  • Status: Working

Events Table Database

  • Description: Database that holds event information such as ID of the user who created event, day, month, year, start time, end time, description, and the eventID
  • Status: Working

Add Event

  • Description: PHP file that adds an event to the event database with all the information from the form submission
  • Status: Working

Delete Event

  • Description: PHP file that deletes an event from the events table based on the eventID
  • Status: Working

Add Account

  • Description: PHP file that adds an account to the account database if the email is not already in use and the required information is present.
  • Status: Working

Log In

  • Description: PHP file that gets the account associate with an email that the user entered. Then checks the password that the user inputted with the password of the retrieved account. If all is correct it sends the user to the calender page
  • Status: Working

Change Password

  • Description: PHP file that modifys the password of a specific users account in the database
  • Status: Working
